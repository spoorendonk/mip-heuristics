# MIPLIB 2017 primal-hard subset — 25 instances (stage 1 of gh#16)
#
# Every instance satisfies both:
#   (a) HiGHS (vanilla, 1 thread, seed 0) does not solve it in 300s, AND
#   (b) finding / improving primal solutions is the dominant difficulty.
#
# Selection procedure:
#   1. Probed ~40 MIPLIB-hard candidates with vanilla HiGHS @ 60s.
#   2. Dropped any that solved in 60s or hit gap < 1% early (none did).
#   3. Dropped two "*-i" instances that use MPS INDICATORS (HiGHS parser fails).
#   4. Picked 25, balanced roughly to Local-MIP Table 1 category ratios.
#
# Probe source data: bench/results_hard_probe_60s/ (re-runnable via
# run_benchmark.py --instances instances_hard_probe.txt --time-limit 60).
#
# Category codes follow Local-MIP §6.1.1 (BP, IP, MBP, MIP).
# Provenance per instance (tag/paper signal → observed 60s behavior):
#   FT   = MIPLIB "feasibility" tag ∩ "hard" tag
#   FPR  = Salvagnin et al. 2025 §6.1 "only one strategy finds feasible"
#   LMR  = Lin, Zou, Cai 2024/25 — new MIPLIB record set by Local-MIP
#   SCY  = Mexi et al. 2023 — hard-LP-relaxation target
#   HT   = generic MIPLIB "hard" tag, picked for category balance
